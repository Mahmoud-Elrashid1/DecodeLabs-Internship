Guidelines

Qualification Criteria

Everything you need to earn your badge and unlock Project 2.

HTML5 Requirements

  • Use semantic landmark elements: header, nav, main, article, footer
  • Every image has a descriptive alt attribute
  • Headings follow a logical h1 → h2 → h3 hierarchy
  • Forms include associated label elements
  • Document has a valid lang attribute on the html tag
  • No div-soup — the right element for the right purpose

CSS3 Requirements

  • Mobile-first: single column base layout
  • Tablet breakpoint at min-width 768px
  • Desktop breakpoint at min-width 1024px
  • CSS Grid for macro page structure
  • Flexbox for component-level alignment
  • clamp() used for fluid, responsive typography
  • CSS custom properties (variables) throughout
  • Max 2 font families, 3 weights — Montserrat + Open Sans

JavaScript Requirements

  • Vanilla JS only — no frameworks or libraries
  • Mobile navigation toggle (hamburger menu)
  • At least one interactive UI component (tabs, accordion, etc.)
  • Smooth scroll behaviour
  • Back-to-top button with scroll visibility logic
  • IntersectionObserver for scroll-triggered animations

Accessibility Requirements

  • Skip-to-content link as the first focusable element
  • All interactive elements reachable and operable by keyboard
  • ARIA roles, labels, and states on custom components
  • Colour contrast ratio minimum 4.5:1 for text
  • Focus indicators visible and clear
  • No content relying on colour alone to convey meaning
